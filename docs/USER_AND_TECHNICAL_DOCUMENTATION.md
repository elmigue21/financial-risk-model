# 4.6 User and Technical Documentation

## User Documentation

### Account Creation and Sign-In

1. **Initial Access:** Open the Financial Health Check web application in your browser.
2. **Registration:** On the sign-in page, click the **"Create an account"** link.
3. **Provide Details:** Enter your Full Name, Email Address, Password, and Confirm Password.
4. **Password Requirements:** Your password must be at least 8 characters and include at least one letter and one number.
5. **Account Setup:** Upon successful registration, you are signed in automatically.
6. **Returning Sign-In:** Existing users sign in with their registered Email Address and Password.
7. **Dashboard Access:** Access the Financial Health Dashboard upon successful authentication.
8. **Session Duration:** You remain signed in for 7 days on that browser; after expiry you are asked to sign in again.
9. **Sign Out:** End your session from the top navigation bar or the Account page.

### Financial Health Dashboard

- **Latest Assessment:** View your most recently recorded month's result.
- **Health Score and Risk Index (0–100):** A simple, intuitive rating with a color-coded band:
  - **Green — Low Risk (0–33):** Healthy financial position
  - **Yellow — Moderate Risk (34–66):** Some areas need attention
  - **Red — High Risk (67–100):** Elevated risk of financial trouble
- **Model Confidence:** The model's certainty in the prediction, shown as a percentage for transparency.
- **Ratio Breakdown:** Each financial ratio with a plain-English status (Healthy, Weak, Tight, Poor, or Critical).
- **Top Risk Factors:** The specific ratios pushing your risk up, taken from the model's own math.
- **AI Insights:** Flags on inputs that look problematic (when the AI advisor is enabled).
- **AI Advisor Chat:** An opening assessment plus a live conversation tailored to your numbers and trend.

### Entering Monthly Finances

- **Month Selection:** Choose the month you are recording.
- **This Month's Income:**
  - **Revenue:** All sales / money earned this month
  - **Expenses:** Everything spent this month, including interest
  - **Interest Paid:** Loan interest included in expenses (enter 0 if none)
- **End-of-Month Balances:**
  - **Cash on Hand:** Money in the bank at month end
  - **Total Assets:** Everything the business owns
  - **Total Liabilities:** Everything the business owes
  - **Owner's Equity:** Your stake (Assets should equal Liabilities + Equity)
- **Live Feedback:** The form shows your net profit and a balance check as you type.
- **Validation:**
  - **Errors** block saving (e.g. negative values, a balance sheet that doesn't add up, or cash greater than total assets)
  - **Warnings** flag likely typos (e.g. no revenue recorded, or an unusually high profit margin)

### Submission Process

- Complete all required fields
- Correct any errors flagged by the form
- Select **"Save & score this month"**
- The system computes your ratios, scores them, and saves the month
- Only one assessment is kept per month — delete a month first to re-enter it

### Viewing Assessments and History

- **History List:** Browse every assessment you've run, newest first.
- **Report Details:** Open any month to view its full report, including:
  - Figures you entered
  - Computed ratios and their statuses
  - Top risk factors
  - AI insights
  - The saved advisor conversation (which you can continue)
- **Export:**
  - **Export PDF:** Download a single report as a formatted PDF file (generated by the app — no print dialog needed).
  - **Export CSV:** Download a single report or your entire history as an Excel-openable CSV.

### AI Financial Advisor

- **Opening Assessment:** A plain-English summary of what your result means.
- **Ask Questions:** Type a question or tap a suggested question.
- **Trend-Aware Advice:** The advisor reads your month-over-month history, not just a single snapshot.
- **Graceful Degradation:** If no AI provider key is configured, scoring, history, and reports still work; the advisor simply stays disabled.

### User Profile Management

- **View Profile:** See your full name, company name, and email address.
- **Update Profile:** Edit your details and save; changing to an email already used by another account is rejected.
- **Change Password:** Enter your current password, then a new password (twice).
- **Confidentiality:** You can only ever access your own financial assessments, reports, and account information.

## Technical Documentation

### Database Schema

The frontend owns all persistence using a local SQLite database (`frontend/data/app.db`).

**Users Table**

```sql
CREATE TABLE users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  created_at    TEXT NOT NULL DEFAULT (datetime('now')),
  full_name     TEXT NOT NULL,
  email         TEXT NOT NULL,
  password_hash TEXT NOT NULL
);

CREATE UNIQUE INDEX users_email ON users(email);
```

**Predictions Table**

```sql
CREATE TABLE predictions (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  created_at    TEXT NOT NULL DEFAULT (datetime('now')),
  user_id       INTEGER,          -- owner (FK -> users.id)
  assessed_on   TEXT,
  month         TEXT,             -- "YYYY-MM"
  figures       TEXT,             -- JSON: raw dollar inputs
  inputs        TEXT NOT NULL,    -- JSON: the 6 computed ratios
  probability   REAL NOT NULL,
  risk_level    TEXT NOT NULL,
  confidence    REAL,
  risk_index    INTEGER NOT NULL,
  contributions TEXT,             -- JSON: per-feature risk contributions
  ai_flags      TEXT,             -- JSON: AI input flags
  messages      TEXT,             -- JSON: advisor conversation
  suggestions   TEXT              -- JSON: suggested questions
);

-- One assessment per month, per user
CREATE UNIQUE INDEX predictions_user_month ON predictions(user_id, month);
```

### API Endpoints

**Authentication Endpoints**

- `POST /api/auth/register` – Create a new account and start a session
- `POST /api/auth/login` – Authenticate with email and password
- `POST /api/auth/logout` – End the current session

**Profile Endpoints**

- `GET /api/profile` – Get the current user's profile
- `PATCH /api/profile` – Update the current user's profile
- `PATCH /api/profile/password` – Change the current user's password

**Assessment and History Endpoints**

- `POST /api/predict` – Score a company (proxies to the Python model service)
- `GET /api/history` – Get the current user's saved assessments
- `POST /api/history` – Save a new assessment
- `GET /api/history/[id]` – Get one full assessment
- `PATCH /api/history/[id]` – Attach AI data (flags, chat, suggestions) to an assessment
- `DELETE /api/history/[id]` – Delete an assessment

**AI Endpoints**

- `POST /api/advice` – Streamed opening assessment and advisor replies
- `POST /api/flag` – Mark inputs as critical / warning / ok
- `POST /api/suggestions` – Generate three follow-up questions

**Model Service Endpoints (Python, port 5000)**

- `GET /health` – Liveness check
- `POST /predict` – Score a company from its financial ratios

### Development Setup

**Prerequisites**

- Node.js 18+ and npm
- Python 3.9+ and pip
- (Optional) Groq API key for the AI advisor

**Installation Steps**

- **Clone Repository**
  - `git clone <repository-url>`
  - `cd financial-risk`
- **Start the Python Model Service**
  - `cd financial-risk-model`
  - `pip install -r requirements.txt`
  - `python app.py`  (runs on http://localhost:5000)
- **Configure and Start the Frontend**
  - `cd frontend`
  - `npm install`
  - `cp .env.example .env.local`
  - Configure the following variables:
    - `AUTH_SECRET="your-generated-secret"`
    - `GROQ_API_KEY="your-groq-api-key"`  (optional)
    - `MODEL_API_URL="http://localhost:5000"`
  - `npm run dev`
- **Access the Application**
  - Open http://localhost:3000

**Notes**

- Generate `AUTH_SECRET` with: `node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"`
- The SQLite database file is created automatically on first run; no separate database server is required.
- For production, run `npm run build` then `npm start`, and deploy the frontend to a Node host with a persistent filesystem (so the SQLite file survives restarts).
